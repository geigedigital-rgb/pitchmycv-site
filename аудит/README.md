# GEO/SEO Audit Bundle for Pitch My CV

This folder contains the external `geo-seo-claude` tool, its instructions, and the audit output for `https://pitchcv.app`.

## Folder contents

- `geo-seo-claude/` - local copy of `zubair-trabzada/geo-seo-claude` with the original instructions, skill files, agents, and Python utilities.
- `GEO-AUDIT-REPORT.md` - final audit report synthesized from the tool's methodology.
- `llms.txt` - recommended draft `llms.txt` file for the site.
- `llms-full.txt` - extended draft `llms-full.txt`.
- `artifacts/` - raw JSON artifacts generated by running the tool's utilities against the live domain.

## Key instruction files inside `geo-seo-claude`

- `geo-seo-claude/README.md` - tool overview, commands, and architecture.
- `geo-seo-claude/geo/SKILL.md` - main orchestrator, full audit methodology, and GEO score formula.
- `geo-seo-claude/skills/geo-audit/SKILL.md` - full audit structure and final report format.
- `geo-seo-claude/agents/geo-ai-visibility.md` - citability, AI crawlers, `llms.txt`, and brand mentions.
- `geo-seo-claude/agents/geo-technical.md` - technical SEO, SSR, headers, and crawlability.
- `geo-seo-claude/agents/geo-content.md` - E-E-A-T, readability, and topical authority.
- `geo-seo-claude/agents/geo-schema.md` - schema validation, `sameAs`, and Person/Organization gaps.
- `geo-seo-claude/agents/geo-platform-analysis.md` - readiness for Google AIO, ChatGPT, Perplexity, Gemini, and Bing Copilot.

## How the audit was executed

The upstream `/geo audit` command is designed to run inside Claude Code as a custom skill. In this repository, the audit was reproduced using the original methodology:

1. copy the upstream repository into this folder as `./geo-seo-claude/`;
2. run the tool's Python utilities against the live site `https://pitchcv.app`;
3. synthesize the final report using the categories defined in `geo/SKILL.md` and `skills/geo-audit/SKILL.md`.

## Commands used

```bash
python3 "./geo-seo-claude/scripts/fetch_page.py" "https://pitchcv.app" full
python3 "./geo-seo-claude/scripts/fetch_page.py" "https://pitchcv.app" sitemap
python3 "./geo-seo-claude/scripts/citability_scorer.py" "https://pitchcv.app"
python3 "./geo-seo-claude/scripts/citability_scorer.py" "https://pitchcv.app/careers/software-engineer-resume"
python3 "./geo-seo-claude/scripts/citability_scorer.py" "https://pitchcv.app/tools/ats-checker"
python3 "./geo-seo-claude/scripts/citability_scorer.py" "https://pitchcv.app/resume-keyword-optimization"
python3 "./geo-seo-claude/scripts/llmstxt_generator.py" "https://pitchcv.app" validate
python3 "./geo-seo-claude/scripts/llmstxt_generator.py" "https://pitchcv.app" generate
python3 "./geo-seo-claude/scripts/brand_scanner.py" "Pitch CV" "pitchcv.app"
```

## Notes

- The report reflects the live domain and public search-accessible signals, not only the local repository files.
- Missing security headers are most likely a deploy/server configuration issue rather than an HTML template issue.
- The `llms.txt` and `llms-full.txt` drafts in this folder are recommendations and are not yet published at the site root.
