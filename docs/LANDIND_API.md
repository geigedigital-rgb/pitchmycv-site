# API для лендинга (pitchcv.app)

Документация по интеграции лендинга **pitchcv.app** с бэкендом и приложением **my.pitchcv.app**.

---

## 1. Целевой флоу (основной)

**На лендинге мы не обрабатываем и не показываем результаты проверки.** Резюме и вакансия только сохраняются на бэкенде; лендинг сразу перенаправляет пользователя в приложение (на страницу логина) с одноразовым токеном. Обработка (анализ резюме) происходит уже **в приложении** после входа: данные подставляются, анализ запускается автоматически.

### Последовательность

1. **Лендинг:** пользователь загружает резюме и вакансию, нажимает кнопку (например, «Проверить резюме» / «Узнать результат»).
2. **Бэкенд:** вызывается **POST /api/landing/save**. Данные сохраняются во временное хранилище, возвращается одноразовый токен. **Анализ на этом этапе не выполняется.**
3. **Редирект:** лендинг перенаправляет на `https://my.pitchcv.app/login?pending=TOKEN`. Результаты на лендинге не показываются — пользователь сразу уходит в приложение.
4. **Логин:** на странице логина показывается **стандартная заставка приложения** (как при обычном входе): левая панель с визуализацией, правая — форма входа (Google / email). Превью содержимого резюме не показывается. При наличии `?pending=TOKEN` приложение запрашивает **GET /api/landing/pending?token=TOKEN** (без авторизации) и под формой входа выводит блок «Ваши файлы готовы к анализу»: только название файла резюме и название вакансии — без усложнений.
5. **После входа:** приложение перенаправляет на `/optimize?pending=TOKEN`, вызывает **GET /api/landing/claim?token=TOKEN** (с авторизацией), получает сохранённые `resume_content` и `job_text`, подставляет их в экран анализа и **сразу автоматически запускает анализ** (POST /api/analyze). Пользователь видит прогресс и затем результат проверки (ATS, ключевые слова, рекомендации) уже в приложении.

### Что делает лендинг, что — приложение

| Где | Действие |
|-----|----------|
| **Лендинг** | Принять файл/текст резюме и вакансию → отправить в **POST /api/landing/save** → получить `token` → редирект на `https://my.pitchcv.app/login?pending=TOKEN`. Никакой обработки результатов, никакого анализа на лендинге. |
| **Приложение** | На логине — стандартная заставка входа; при `?pending=TOKEN` — блок «Файлы готовы» (имя файла + вакансия). После входа — **GET /api/landing/claim** → подстановка в форму и автоматический запуск анализа → результат и флоу (оптимизация после триала и т.д.). |

### Продуктовые правила

- **Одна бесплатная проверка** на пользователя (с антифродом): только **анализ** (ATS score, ключевые слова, рекомендации). Без оптимизации.
- **Оптимизация** (генерация улучшенного PDF) доступна **только после триала/покупки**. Кнопка «Оптимизировать» на бесплатном тарифе ведёт на покупку (триал или подписка).
- **Маркетинговый флоу** между результатом проверки и покупкой нужно выстроить явно:
  - Показать результат бесплатного анализа (оценка, что улучшить).
  - Чёткий призыв к действию: «Улучшить резюме под вакансию», «Скачать готовое резюме под ATS» и т.п.
  - Кнопка ведёт на страницу апгрейда/триала; после оплаты — доступна кнопка «Оптимизировать» и скачивание PDF.

## 2. Сохранить и перейти на логин: POST /api/landing/save

**POST** `https://my.pitchcv.app/api/landing/save`  
(Локально: `http://localhost:8000/api/landing/save`)

**Content-Type:** `multipart/form-data`

### Параметры (form fields)

| Поле         | Тип    | Обязательное | Описание |
|-------------|--------|---------------|----------|
| `resume`    | file   | да*           | Файл резюме: PDF, DOCX или текстовый файл. Макс. 5 МБ. |
| `resume_text` | string | да*         | Альтернатива файлу — резюме вставлено текстом. Макс. 50 000 символов. |
| `job_text`  | string | да            | Текст вакансии, вставленный вручную. |

\* Нужно одно из двух: либо `resume`, либо `resume_text`.  
\** Для вакансии используется только `job_text`.

### Ответ (200 OK)

```json
{
  "token": "abc123...",
  "resume_filename": "resume.pdf",
  "job_title": "Software Engineer"
}
```

- **token** — подставить в URL редиректа: `https://my.pitchcv.app/login?pending=TOKEN`. TTL токена: `LANDING_PENDING_TTL_SECONDS` (по умолчанию 15 минут).
- **resume_filename**, **job_title** — можно кратко показать на лендинге перед редиректом (опционально), например: «Перенаправляем в личный кабинет…» Без вызова analyze и без отображения результатов проверки.

### Что происходит после редиректа (логин и дальше)

1. **Страница логина** (`/login?pending=TOKEN`): приложение вызывает **GET /api/landing/pending?token=TOKEN** (без авторизации) и показывает блок «Ваши файлы готовы к анализу» с названием файла и названием вакансии.
2. **Пользователь входит** (Google или email/password).
3. **После успешного входа** приложение вызывает **GET /api/landing/claim?token=TOKEN** (с заголовком Authorization). Токен при этом потребляется (одноразовый). В ответе: `resume_content`, `job_text`, `resume_filename`.
4. **Данные подставляются** в экран анализа/обработки (Optimize или аналог), и **сразу автоматически запускается анализ** (POST /api/analyze). Пользователь видит прогресс, затем результат проверки (ATS, ключевые слова, рекомендации) уже в приложении.
5. Дальше — по продукту: одна бесплатная проверка; оптимизация и скачивание PDF после триала/покупки.

### Ошибки

| Код | Причина |
|-----|--------|
| 400 | Нет резюме или вакансии, пустой/слишком длинный текст. |
| 503 | Сервис временно недоступен или не настроен GOOGLE_API_KEY. |

Rate limit по IP к **save** не применяется (лимит «одна бесплатная проверка» учитывается уже в приложении после входа).

## 3. Бесплатная проверка без входа (опционально): POST /api/landing/analyze

Использовать **только** если нужен сценарий «результат прямо на лендинге без логина». В целевом флоу результат на лендинге не показываем — используется только **save** и редирект на логин.

**POST** `https://my.pitchcv.app/api/landing/analyze`

**Content-Type:** `multipart/form-data`  
Параметры — те же, что у `/api/landing/save`: `resume` или `resume_text`, и обязательный `job_text`.

### Ответ (200 OK)

JSON в формате `/api/analyze`: `ats_score`, `keyword_score`, `job`, `recommendations`, `improvement_tips`, `skills_score`, `experience_score`, `portfolio_score`.

### Ошибки

Те же 400, 422, 503; дополнительно **429** — лимит: одна проверка на IP за N часов (по умолчанию 24). Сообщение: «Одна бесплатная проверка на 24 часа. Войдите на my.pitchcv.app для проверки с сохранением результата.»

## 4. CORS

Бэкенд принимает запросы только с доменов из **LANDING_ALLOWED_ORIGINS** (см. раздел «Настройка бэкенда»). Добавьте:

- `https://pitchcv.app`
- `https://www.pitchcv.app`

Для локальной разработки при необходимости: `http://localhost:PORT`.

---

## 5. Антифрод и лимиты

- **Одна бесплатная проверка (анализ)** на пользователя — с антифродом (в приложении после входа). Оптимизация (генерация PDF) только после триала/покупки.
- Для **POST /api/landing/analyze** (если используется): один запрос на IP за N часов (по умолчанию 24). In-memory на бэкенде; при нескольких инстансах — Redis (можно доработать).
- **Максимальный размер резюме:** 50 000 символов после извлечения из PDF/DOCX (`LANDING_MAX_RESUME_CHARS`).
- **Максимальный размер файла резюме:** 5 МБ.
- Рекомендации на лендинге: honeypot, при необходимости reCAPTCHA перед отправкой.

## 6. Пример запроса с лендинга (основной флоу: save → редирект)

```javascript
const formData = new FormData();
formData.append("resume", fileInput.files[0]);  // или formData.append("resume_text", textarea.value);
formData.append("job_text", jobTextarea.value);

const response = await fetch("https://my.pitchcv.app/api/landing/save", {
  method: "POST",
  body: formData,
  credentials: "omit",
});

if (!response.ok) {
  const err = await response.json().catch(() => ({}));
  throw new Error(err.detail || response.statusText);
}

const { token, resume_filename, job_title } = await response.json();
// Сразу редирект на логин — результат не показываем на лендинге
window.location.href = `https://my.pitchcv.app/login?pending=${encodeURIComponent(token)}`;
```

### Пример curl (save)

```bash
curl -X POST "https://my.pitchcv.app/api/landing/save" \
  -F "resume=@/path/to/resume.pdf" \
  -F "job_text=Full job description text..."
```

## 7. Настройка бэкенда (.env)

```env
GOOGLE_API_KEY=...

LANDING_ALLOWED_ORIGINS=https://pitchcv.app,https://www.pitchcv.app
LANDING_RATE_LIMIT_HOURS=24
LANDING_MAX_RESUME_CHARS=50000
LANDING_PENDING_TTL_SECONDS=900
```

После изменения `LANDING_ALLOWED_ORIGINS` бэкенд перезапустить.

## 8. Что показывать на лендинге (рекомендации)

- **Одна основная кнопка** после ввода резюме и вакансии: «Проверить резюме» / «Узнать результат» → вызов **POST /api/landing/save** и редирект на `https://my.pitchcv.app/login?pending=TOKEN`. Результат на лендинге не показываем.
- Кратко перед редиректом можно показать: «Перенаправляем вас в личный кабинет для просмотра результата» (и при желании `resume_filename`, `job_title`).
- Ссылку на приложение: **https://my.pitchcv.app**.

## 9. Флоу в приложении после входа (для реализации фронта my.pitchcv.app)

1. После успешного логина с `?pending=TOKEN`: вызвать **GET /api/landing/claim?token=TOKEN** (с авторизацией).
2. Получить `resume_content`, `job_text`, `resume_filename`; подставить в форму на экране анализа/обработки.
3. **Сразу запустить анализ** (POST /api/analyze или эквивалент), не дожидаясь дополнительных действий пользователя. Показать прогресс и по завершении — результат (ATS, ключевые слова, рекомендации).
4. На бесплатном тарифе: **не показывать** кнопку «Оптимизировать» как доступную; вместо неё — CTA на апгрейд/триал («Улучшить резюме под вакансию», «Скачать PDF под ATS» → переход на страницу покупки).
5. После триала/покупки: кнопка «Оптимизировать» активна, доступно скачивание сгенерированного PDF.

## 10. Сводка для фронта лендинга

- **Основной флоу:** пользователь нажимает кнопку проверки → **POST /api/landing/save** (поля: `resume` или `resume_text`, плюс `job_text`) → ответ `{ token, resume_filename, job_title }` → редирект на **https://my.pitchcv.app/login?pending=TOKEN**. Результат на лендинге не показываем.
- После входа пользователь попадает в приложение на экран обработки с автоматически запущенным анализом. Одна бесплатная проверка (анализ) с антифродом; оптимизация только после триала; флоу «результат проверки → CTA на улучшение → покупка» выстраивается маркетингово в приложении.
- Опционально: **POST /api/landing/analyze** — результат сразу в JSON (для сценария «проверка без входа»); лимит 1 запрос на IP за 24 часа.

## 11. Настройки для подключения лендинга (передать команде лендинга)

Ниже — всё, что нужно для интеграции лендинга с приложением.

### Базовый URL API и приложения

- **Приложение (логин после редиректа):** `https://my.pitchcv.app`
- **API бэкенда:** `https://my.pitchcv.app/api` (тот же хост; приложение и API раздаются с одного домена или за одним прокси).

Если бэкенд на другом хосте (например, отдельный сервер), замените базовый URL API в примерах ниже.

### Endpoint для основного флоу (обязательно)

| Метод | URL | Назначение |
|-------|-----|------------|
| POST | `{API}/landing/save` | Сохранить резюме и вакансию, получить токен. После ответа — редирект на `https://my.pitchcv.app/login?pending={token}`. |

**Тело запроса:** `multipart/form-data`.

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| `resume` | file | да* | Файл резюме (PDF, DOCX или текст). Макс. 5 МБ. |
| `resume_text` | string | да* | Резюме текстом (альтернатива файлу). Макс. 50 000 символов. |
| `job_text` | string | да | Текст вакансии. |

\* Нужно одно из двух: `resume` или `resume_text`.  
\** Для вакансии используется только `job_text`.

**Ответ 200:**  
`{ "token": "...", "resume_filename": "...", "job_title": "..." }`  
→ редирект: `https://my.pitchcv.app/login?pending={token}`.

### CORS

Бэкенд разрешает запросы только с доменов, перечисленных в настройках (например, `https://pitchcv.app`, `https://www.pitchcv.app`). Для локальной разработки при необходимости добавляют `http://localhost:PORT`. Список доменов настраивается на бэкенде (переменная `LANDING_ALLOWED_ORIGINS`).

### Пример кода на лендинге (основной флоу)

```javascript
const formData = new FormData();
formData.append("resume", fileInput.files[0]);   // или formData.append("resume_text", textarea.value);
formData.append("job_text", jobTextarea.value);

const response = await fetch("https://my.pitchcv.app/api/landing/save", {
  method: "POST",
  body: formData,
  credentials: "omit",
});

if (!response.ok) {
  const err = await response.json().catch(() => ({}));
  throw new Error(err.detail || response.statusText);
}

const { token } = await response.json();
window.location.href = `https://my.pitchcv.app/login?pending=${encodeURIComponent(token)}`;
```

### Что видит пользователь после редиректа

1. **Страница логина** — стандартный экран входа приложения (без превью резюме). При наличии `?pending=TOKEN` под формой показывается блок «Ваши файлы готовы»: имя файла резюме и вакансия.
2. После входа (Google или email) — переход на экран анализа, данные подставляются, анализ запускается автоматически. Результат (ATS, ключевые слова, рекомендации) отображается в приложении.
